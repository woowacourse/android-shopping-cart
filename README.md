  - [ ] 전체 데이터 개수보다 큰 범위(남은 데이터가 20개 미만)를 요청했을 때, 예외 없이 남은 개수만큼만 정상 반환하는가?
- **페이지네이션 기반 로딩 (`CartRepository`)**:
  - [ ] 특정 페이지 번호와 사이즈(5개)를 요청했을 때, 해당 페이지에 맞는 올바른 인덱스의 데이터를 반환하는가?

## 🧪 테스트 명세

### 비즈니스 로직 단위 테스트
- **금액 값 객체(`Money`)**:
    - [x] 0 이상의 정수가 주어지면 `Money` 객체가 정상적으로 생성된다.
    - [x] 음수의 정수로 생성하려고 할 경우 적절한 예외가 발생한다.
    - [x] 동일한 금액을 가진 두 `Money` 객체는 동등한 것으로 간주한다.
- **일급 컬렉션(`Products`)**:
    - [x] 유효한 상품 데이터 리스트가 주어지면 `Products` 객체가 정상적으로 생성된다.
    - [x] 생성된 `Products` 객체에서 내부의 전체 상품 데이터를 누락이나 변형 없이 정상적으로 반환한다.
- **장바구니(`Cart`) 상태 관리 로직**:
    - [x] 정상적인 상품 객체를 장바구니에 추가하면, 장바구니 내부 목록에 해당 상품이 올바르게 반영된다.
    - [x] 장바구니에 존재하는 상품을 삭제 요청하면, 목록에서 해당 상품이 정상적으로 제거된다.
    - [x] 존재하지 않는 상품의 삭제를 시도할 경우, 상태가 변경되지 않거나 올바르게 무시/예외 처리된다.

    - [ ] 각 아이템 컴포넌트가 Coil을 호출하여 URL 기반 이미지를 정상적으로 로드한다.
