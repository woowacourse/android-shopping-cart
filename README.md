## 🚀 2단계 - 데이터 로딩

## 2단계 기능 및 작업 명세

### 상품 목록 더보기 구현

- 상품 목록은 최초에 20개만 표시한다.
- 목록 하단의 더보기 버튼을 누르면 20개씩 추가로 로드된다.
- 더 로드할 상품이 없으면 더보기 버튼을 숨긴다.
- `ProductsStateHolder`는 현재까지 로드된 상품 목록과 추가 로드 가능 여부(`hasMore`)를 상태로 노출한다.

### 장바구니 페이지네이션 구현

- 장바구니 목록은 5개 단위로 페이지를 나눠 표시한다.
- 화면 하단에 `< 페이지 번호 >` 형식의 페이지 컨트롤을 표시한다.
- 첫 번째 페이지에서는 이전(`<`) 버튼을 비활성화한다.
- 마지막 페이지에서는 다음(`>`) 버튼을 비활성화한다.
- `CartStateHolder`는 현재 페이지의 항목, 현재 페이지 번호(`currentPage`), 첫 페이지 여부(`isFirstPage`), 마지막 페이지 여부(`isLastPage`)를 상태로 노출한다.

### 데이터 로딩 전략 구현

- `ProductRepository`에 페이지 기반 상품 조회 메서드를 추가한다. (`getProducts(page, pageSize)`)
- `CartRepository`에 페이지 기반 장바구니 조회 메서드를 추가한다. (`getCartItems(page, pageSize)`)

## 2단계 테스트 명세

### 상품 페이지네이션 단위 테스트

- 첫 번째 페이지 요청 시 첫 20개 상품이 반환된다.
- 두 번째 페이지 요청 시 다음 20개 상품이 반환된다.
- 마지막 페이지 이후 요청 시 빈 목록이 반환된다.
- 전체 상품 수보다 적은 상품이 남았을 때 남은 상품만 반환된다.

### 장바구니 페이지네이션 단위 테스트

- 첫 번째 페이지 요청 시 첫 5개 항목이 반환된다.
- 마지막 페이지에서는 남은 항목만 반환된다.
- 페이지 범위를 벗어난 요청 시 빈 목록이 반환된다.

## 🚀 1단계 - 상품 목록

안드로이드 8기 레벨 1 미션, 상품 목록과 장바구니를 관리하는 프로젝트입니다.

## 1단계 기능 및 작업 명세

### 상품 목록 화면 구현

- 상품 목록을 격자 형태로 표시한다.
- 각 상품에는 이미지, 상품명, 가격이 표시된다.
- 상품을 클릭하면 상품 상세 화면으로 이동한다.
- 상단에 장바구니 화면으로 이동할 수 있는 버튼이 있다.

### 상품 상세 화면 구현

- 상품의 이미지, 상품명, 가격을 표시한다.
- 장바구니에 상품을 담을 수 있는 버튼이 있다.
- 화면을 닫으면 상품 목록으로 돌아간다.

### 장바구니 화면 구현

- 담긴 상품의 이미지, 상품명, 가격을 목록으로 표시한다.
- 각 항목을 개별 삭제할 수 있다.

### 상태 홀더 구현

- 상태 홀더는 도메인 모델을 UI 모델로 변환하여 화면에 전달한다.

### 상품 도메인 구현

- 상품은 이미지 URL, 상품명, 가격으로 구성된다.
- 상품명은 빈 값이나 공백만으로 이루어질 수 없다.
- 가격은 음수일 수 없다.
- 상품 목록은 중복된 상품을 허용하지 않는다.

### 장바구니 도메인 구현

- 장바구니 항목은 상품과 수량으로 구성된다.
- 수량은 반드시 1개 이상이어야 한다.
- 이미 담긴 상품을 다시 담으면 수량이 증가한다.
- 장바구니 전체 금액을 계산할 수 있다.

## 테스트 명세

### 단위 테스트

- 상품명이 빈 값이거나 공백인 경우 생성이 불가능하다.
- 가격이 음수인 경우 생성이 불가능하다.
- 수량이 1 미만인 경우 생성이 불가능하다.
- 동일한 상품이 목록에 중복으로 추가되지 않는다.
- 이미 담긴 상품을 장바구니에 추가하면 신규 항목이 추가되지 않는다.
