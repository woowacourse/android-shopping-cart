## 🧪 테스트 명세

### 비즈니스 로직 단위 테스트
- **금액 값 객체(`Money`)**:
    - [x] 0 이상의 정수가 주어지면 `Money` 객체가 정상적으로 생성된다.
    - [x] 음수의 정수로 생성하려고 할 경우 적절한 예외가 발생한다.
    - [x] 동일한 금액을 가진 두 `Money` 객체는 동등한 것으로 간주한다.
- **일급 컬렉션(`Products`)**:
    - [ ] 유효한 상품 데이터 리스트가 주어지면 `Products` 객체가 정상적으로 생성된다.
    - [ ] 생성된 `Products` 객체에서 내부의 전체 상품 데이터를 누락이나 변형 없이 정상적으로 반환한다.
- **장바구니(`Cart`) 상태 관리 로직**:
    - [ ] 정상적인 상품 객체를 장바구니에 추가하면, 장바구니 내부 목록에 해당 상품이 올바르게 반영된다.
    - [ ] 장바구니에 존재하는 상품을 삭제 요청하면, 목록에서 해당 상품이 정상적으로 제거된다.
    - [ ] 존재하지 않는 상품의 삭제를 시도할 경우, 상태가 변경되지 않거나 올바르게 무시/예외 처리된다.
- **데이터 저장소(`Repository`)**:
    - [ ] `ProductRepository` 인터페이스의 구현체를 통해 상품 목록을 성공적으로 불러오는지 확인한다.
    - [ ] `CartRepository` 인터페이스의 구현체를 통해 장바구니 상태 변화(추가/삭제)가 데이터 소스에 올바르게 반영되는지 확인한다.

### UI 상태(State) 테스트
- **상품 목록 화면**:
    - [ ] 제공된 상품 데이터 리스트의 크기만큼 `LazyVerticalGrid` 내에 아이템이 누락 없이 렌더링된다.
    - [ ] 각 아이템 컴포넌트가 Coil을 호출하여 URL 기반 이미지를 정상적으로 로드한다.
- **상품 상세 화면 및 네비게이션**:
    - [ ] 리스트에서 특정 상품을 클릭하면, 선택된 상품의 식별자를 기반으로 올바른 상세 정보 상태로 전환된다.
